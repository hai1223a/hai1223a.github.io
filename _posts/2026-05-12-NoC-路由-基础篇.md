路由可以分为确定性路由（deterministic routing）、遗忘路由（oblivious routing）和自适应路由（adaptive routing）。其中 dimension-ordered routing（DOR）是 NoC 中最常用的确定性路由。以 mesh 结构为例，消息从 (0,0) 到 (2,3)，确定性路由 X-Y DOR 先走 X，再走 Y；遗忘路由可以在 X-Y 或 Y-X 之间预先选择；自适应路由在走到 (1,0) 后，如果发现拥塞，就可以换方向。

![image-20260511155003956](/images/18.jpg)
<p class="image-caption">图 1：DOR、遗忘路由与自适应路由示例。DOR 展示了 mesh 中从 (0,0) 到 (2,3) 的 X-Y 路径；遗忘路由展示了同一源宿对之间可在发送前预先选择的 X-Y 与 Y-X 两条路径；自适应路由展示了在 (1,0) 遇到拥塞时偏离 X-Y 路径的一种可能路径。</p>

路由还可以分为最小路径路由（minimal routing）和非最小路径路由（non-minimal routing）。不考虑阻塞时，前者更好；考虑阻塞时，后者可能更好，因为它可以通过绕路避开拥挤区域。

## 确定性路由（DOR）

路由算法可以被描述为哪些 turn 是允许的。比如在 2D mesh 中，下图展示了所有可能的 turn，例如最左下角表示向西传输的信息允许向北转弯。但是如果被允许的 turn 能够组合成环，就会出现死锁，所以路由算法可以通过禁止部分 turn 来实现无死锁路由。

![image-20260511161927041](/images/19.jpg)
<p class="image-caption">图 2：2D Mesh 中可能出现的路由转向</p>

确定性路由是指两节点间的路由路径是固定的。右图展示了 X-Y DOR，它不允许向北或向南传输的信息再转向，所以实现上会先走 X 方向，再走 Y 方向。Y-X DOR 与之类似。但这种算法消除了 mesh 中的路径多样性，会降低吞吐率。

## 遗忘路由（Valiant's Randomized Routing）

遗忘路由虽然允许两节点间存在不同的路由路径，但是不会考虑网络的实时状态，所以实现比较简单。Valiant's randomized routing 是一种经典遗忘路由。Valiant 的思想是针对 $s \to d$ 的通信，随机选择一个 $d'$ 中间节点，完成 $s \to d' \to d$ 的通信，如左图所示。

为了避免 Valiant 路由牺牲通信局部性，可以添加 minimal 约束，$d'$ 的选择只能落在 $s$ 和 $d$ 构成的矩形范围内，如右图所示。

![image-20260511163442138](/images/20.jpg)
<p class="image-caption">图 3：遗忘路由示例</p>

如果将 Valiant 路由或者约束后的 Valiant 路由与 X-Y DOR 结合，即 $s \to d'$ 和 $d' \to d$ 都使用 X-Y DOR，那么该路由算法就是无死锁的。

> *需要注意，还有一种遗忘路由是随机选择 X-Y DOR 或 Y-X DOR，这类路由算法是存在死锁风险的。*

## 自适应路由

自适应路由利用网络信息选择路由路径，信息通常来自本地路由器。自适应路由可以约束为 minimal，也可以设计为允许错误路由，即 non-minimal。若允许 non-minimal，可能会出现活锁，即信息不断被错误路由，导致其永远无法到达目的地。自适应路由的另一个挑战是可能需要保证到达目标节点的多个数据包的一致性。

> 解决活锁的方法：设置每个数据包的最大错误路由数，并给予错误路由次数多的数据包更高的优先级
>
> 解决数据包一致性问题：在目的端重排序；限制某一类保序数据包的路由

死锁同样是自适应路由的一个问题。在 **确定性路由** 一节中提到了 turn 模型，这里进行衍生得到自适应 turn 模型，通过保留 mesh 网络 8 个 turn 中的 6 个实现无死锁自适应路由。

1. **第一种**不允许数据包向西转向，使用时必须先完成向西的传输。
2. **第二种**数据包向北传输后不允许转向，使用时必须最后进行向北的传输。
3. **第三种**为**Negative-First algorithm**，使用时数据包必须先完成负方向（向南和向西）的传输，才能完成正方向（向北和向东）的传输。

![image-20260511173228801](/images/21.jpg)
<p class="image-caption">图 4：Turn model routing</p>

以上三种路由是无死锁的，但下面这个路由是存在死锁的。

![image-20260511174112132](/images/22.jpg)
<p class="image-caption">图 5：Turn model deadlock</p>

**Odd-even turn model** 也是一种适合 mesh 的自适应路由，其按照 **列号 x 是奇数列还是偶数列** 来限制转向。

```
偶数列：不能从东向进入后再转向北/南
奇数列：不能从北/南进入后再转向西
```

## 广播和多播

点对点的通信叫做**单播（unicast）**，在很多场景下，一个 core 需要把同一条消息发送给多个 core。这种通信称为 **广播（broadcast）** 或 **多播（multicast）**。最朴素的实现便是对每个目标节点发送一次 unicast，但效果并不好。

> 广播（broadcast）是指系统中所有的core都需要接收到该消息
>
> 多播（multicast）是指系统中有多个core需要接收到该消息

### Multicast：Virtual Circuit Tree Multicasting（VCTM）

**VCTM** 的思想是：先建立一棵多播树，再让后续多播数据沿着这棵树转发。

假设源节点 `S` 要把同一条消息发给 `D1、D2、D3`。如果直接用 unicast，需要发送三份数据：

```
S -> D1
S -> D2
S -> D3
```

这样会产生大量重复流量。VCTM 的做法是把过程分成两步。

第一步是 **setup 阶段**。源节点分别向 `D1、D2、D3` 发送 setup packet。这些 setup packet 按照普通 XY 路由前进，并在沿途路由器中记录转发方向。属于同一组目标集合 `{D1, D2, D3}` 的 setup packet 使用同一个 **VCT ID**。

例如某个路由器最终记录到：

```
VCT ID = 5 -> East, North
```

这表示：以后携带 `VCT ID = 5` 的 multicast flit 到达这个路由器时，需要被复制并分别转发到 `East` 和 `North` 两个方向。

第二步是 **数据传输阶段**。真正的 multicast flit 不再分别发送三份，而是携带对应的 `VCT ID` 进入网络。每个路由器查表决定转发方向，如果表项中有多个方向，就在该路由器处分叉复制。

因此，VCTM 可以理解为：

> 用 setup packet 预先在路由器里“铺好”一棵多播树，之后 multicast flit 只需要带着 VCT ID，就可以沿树自动分叉转发。

它适合目标集合会被重复使用的 multicast 场景。代价是路由器需要额外的 VCT 表，并且要支持 flit forking。

------

### Broadcast：Whirl

**Whirl** 面向的是 broadcast，也就是一个源节点要把消息发送给所有节点。

它的思想是：**不要每次广播都走同一棵固定树，而是动态生成不同的 broadcast tree，让流量分布到更多链路上。**

如果每次广播都使用固定路径，那么某些链路会长期成为瓶颈：

```
S
├── A
│   ├── B
│   └── C
└── D
    ├── E
    └── F
```

Whirl 会尝试让不同广播使用不同的扩散路径：

```
第 1 次 broadcast：使用树 T1
第 2 次 broadcast：使用树 T2
第 3 次 broadcast：使用树 T3
```

这样可以避免所有广播流量都压在同一批链路上，从而提高链路利用率和吞吐率。

## 不规则拓扑

**Up\*/Down\* routing** 是一种常用于 **不规则拓扑** 的无死锁路由算法。做法是：先从网络中选一个根节点，构造一棵生成树，然后根据这棵树给链路标记方向。朝根节点方向走的链路称为 **up link**，远离根节点方向走的链路称为 **down link**。路由时，消息允许先经过若干条 up link，再经过若干条 down link，即路径**一旦开始向下走，就不能再向上走**。因此，`down -> up` 这种转向是被禁止的。

一个简单例子如下：

```
                R
              /   \
             A     B
           /  \     \
          C    D     E

方向定义：
C -> A -> R        是 up
R -> B -> E        是 down

合法路径：
C -> A -> R -> B -> E
   up   up   down down

D -> A -> C
   up   down

非法路径：
A -> D -> A -> R
   down  up
因为出现了 down -> up
```

它的优点是通用性强，适合不规则网络，并且能够保证 deadlock-free；缺点是**路由表开销大**，需存储完整路由表，其大小与网络节点数的平方成正比，可扩展性差；**路径可能非最短**，因必须遵守"Up-to-Down"的规则，有时不得不绕行；且**负载不均衡**，靠近根节点的链路容易成为瓶颈。

## 路由算法的实现方法

### Source Routing：源路由

**核心在于源节点在 packet header 中直接写入完整路径，中间路由器只按 header 指示转发**。因此 router 简单，每跳延迟低。但是需要源节点和 packet header 都维护不固定长度的路径信息，可扩展性差，并且难以根据拥塞进行路径调节。不过可以通过在源节点为同一目标节点维护多条路径来改善负载均衡。

<img src="/images/23.jpg" alt="Source routing table" style="zoom: 50%;" />
<p class="image-caption">图 6：2x3 mesh 中节点 (0,0) 的源路由表</p>

**例子：**

从源节点到目的节点需要走：

```
East -> East -> North -> North -> Eject
```

那么 packet header 中直接存：

```
<E, E, N, N, Eject>
```

每经过一个 router，就取出最左边的方向并删除：

```
当前 router 读到 E -> 向 East 转发
下一个 router 读到 E -> 继续向 East 转发
```

------

### Node-based Routing Table：逐跳查表路由

**核心在于 packet 只携带目的节点 ID，每个 router 维护一张路由表，根据目的节点查表决定下一跳。**这种方法每个节点的路由表更小，并且能够根据拥塞信息实现一定的自适应。但是每一跳都需要查表，会增加每一跳的延迟。

<img src="/images/24.jpg" alt="Table-based routing" style="zoom:50%;" />
<p class="image-caption">图 7：3x3 mesh 中基于 west-first turn model algorithm 的逐跳查表路由</p>

**例子：**

某个 router 的 routing table 如下：

```
目的节点 D1 -> East
目的节点 D2 -> North
目的节点 D3 -> East 或 North
本节点       -> Eject
```

当 packet 到达该 router，目的地是 `D3`，router 查表发现可以走：

```
East 或 North
```

如果 East 较拥塞，可以选择 North。

------

### Combinational Logic Routing：组合逻辑路由

**核心在于 packet 携带目的节点坐标或 ID，router 用组合逻辑直接计算下一跳，不查表。**该方法通常绑定一种拓扑与一种路由，不具备通用性，也不太适合不规则拓扑。但是它很适合 mesh，实现简单、延迟低，并且能够利用拥塞信息实现自适应。

**例子：**

在 2D mesh 中使用 XY routing：

```
当前节点：(1, 2)
目的节点：(3, 4)

if cur_x < dst_x: go East
else if cur_x > dst_x: go West
else if cur_y < dst_y: go North
else if cur_y > dst_y: go South
else: Eject
```

router 比较坐标：

```
当前 x = 1，目的 x = 3
所以先向 East 走
```

等 x 坐标相同后，再比较 y 坐标：

```
当前 y < 目的 y
所以向 North 走
```

### Congestion-aware / Adaptive Routing：拥塞感知自适应路由

这算是一种自适应的思想，可以与逐跳查表路由、组合逻辑路由进行结合。**核心在于 router 不只看目的地，还结合网络状态，例如 buffer occupancy、credit、故障信息等，动态选择下一跳。**

**例子：**

packet 当前可以向 East 或 North 走：

```
East buffer occupancy  = 8
North buffer occupancy = 2
```

说明 East 更拥塞，于是 router 选择：

```
go North
```
